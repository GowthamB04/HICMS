1. OVERVIEW
================================================================================

This is a full-stack web application with:
- BACKEND: Spring Boot REST API (Java 17, Spring Data JPA, MySQL, Hibernate)
- FRONTEND: Angular 21 Single Page Application
- DATABASE: MySQL 8.0
- PORT (Backend): 8989
- PORT (Frontend dev): 4200
- COMMUNICATION: HTTP/REST with Basic Authentication + JSON payloads

The frontend and backend communicate via HTTP/HTTPS over REST API endpoints
prefixed with /api. During development, the Angular dev server proxies all
/api requests to the backend running on port 8989.

================================================================================
2. COMMUNICATION FLOW
================================================================================

STEP-BY-STEP DATA FLOW:

   User clicks button in Angular component
          ↓
   Component calls ApiService method (e.g., this.apiService.get('/treatments'))
          ↓
   ApiService prepends baseUrl='/api' → '/api/treatments'
          ↓
   ApiService adds Basic Auth header from AuthService
          ↓
   HttpClient sends HTTP request to browser origin (http://localhost:4200)
          ↓
   Angular dev server proxy (proxy.conf.json) intercepts /api/treatments
          ↓
   Proxy forwards request to backend: http://localhost:8989/api/treatments
          ↓
   Backend TreatmentController receives GET request at endpoint /api/treatments
          ↓
   Controller validates Basic Auth credentials via Spring Security
          ↓
   Service layer calls Repository (JPA) to query MySQL database
          ↓
   Database returns data → Service → Controller builds JSON response
          ↓
   Controller returns ResponseEntity with JSON: { message, status, data }
          ↓
   Proxy forwards response back to browser
          ↓
   ApiService Observable emits data to component
          ↓
   Component receives data and renders it in the UI


AUTHENTICATION FLOW:

   User enters username/password in login form
          ↓
   LoginComponent calls ApiService.post('/auth/login', {username, password})
          ↓
   Backend returns: { message, status, token, role, userId, fullName, email }
          ↓
   AuthService stores credentials in localStorage/sessionStorage as Base64
          ↓
   For all subsequent requests, ApiService reads credentials and adds header:
       Authorization: Basic <base64(username:password)>
          ↓
   Backend Spring Security validates this header on each request
          ↓
   If valid, request proceeds; if invalid, returns 401 Unauthorized

     @RestController
     @RequestMapping("/api/treatments")
     @CrossOrigin(origins = "http://localhost:4200", allowCredentials = "true")
     public class TreatmentController {
         @Autowired
         private TreatmentService treatmentService;
         
         @GetMapping
         public ResponseEntity<Map<String, Object>> getAllTreatments() {
             try {
                 List<Treatment> treatments = treatmentService.getAllTreatments();
                 Map<String, Object> response = new HashMap<>();
                 response.put("message", "Treatments fetched successfully");
                 response.put("status", true);
                 response.put("data", treatments);
                 return ResponseEntity.ok(response);
             } catch (Exception e) {
                 Map<String, Object> response = new HashMap<>();
                 response.put("message", "Error fetching treatments: " + e.getMessage());
                 response.put("status", false);
                 return ResponseEntity.status(HttpStatus.INTERNAL_SERVER_ERROR).body(response);
             }
         }
     }
   
   USE: 
   - @RestController: Marks class as a REST API endpoint handler
   - @RequestMapping("/api/treatments"): All methods respond to /api/treatments base path
   - @CrossOrigin(...): Allows frontend on localhost:4200 to access these endpoints
   - @GetMapping, @PostMapping, @PutMapping, @DeleteMapping: HTTP method routing
   - ResponseEntity<Map>: Returns JSON response with standardized format

📄 frontend/insurance-frontend/src/app/services/api.service.ts

     @Injectable({ providedIn: 'root' })
     export class ApiService {
       private baseUrl = '/api';
     
       constructor(private http: HttpClient, private auth: AuthService) {}
     
       private getAuthHeaders(): HttpHeaders {
         const credentials = this.auth.credentials;
         if (credentials) {
           console.log('Sending API request with Basic Auth credentials');
           return new HttpHeaders({
             Authorization: `Basic ${credentials}`,
             'Content-Type': 'application/json',
           });
         }
         console.warn('No credentials found for API request');
         return new HttpHeaders({
           'Content-Type': 'application/json',
         });
       }
     
       get<T>(path: string): Observable<T> {
         const headers = this.getAuthHeaders();
         console.log('GET request to:', this.baseUrl + path);
         return this.http.get<T>(`${this.baseUrl}${path}`, { headers });
       }
     
       post<T>(path: string, body: any): Observable<T> {
         const headers = this.getAuthHeaders();
         console.log('POST request to:', this.baseUrl + path);
         return this.http.post<T>(`${this.baseUrl}${path}`, body, { headers });
       }
     
       put<T>(path: string, body: any): Observable<T> {
         const headers = this.getAuthHeaders();
         console.log('PUT request to:', this.baseUrl + path);
         return this.http.put<T>(`${this.baseUrl}${path}`, body, { headers });
       }
     
       delete<T>(path: string): Observable<T> {
         const headers = this.getAuthHeaders();
         console.log('DELETE request to:', this.baseUrl + path);
         return this.http.delete<T>(`${this.baseUrl}${path}`, { headers });
       }
     }
   
   USE:
   - Wraps Angular HttpClient to centralize API communication
   - baseUrl = '/api': All API calls start with /api (handled by proxy)
   - getAuthHeaders(): Reads stored credentials and adds Authorization header
   - get/post/put/delete: Helper methods for CRUD operations
   - Observable<T>: RxJS observables for async HTTP calls
   
   EXAMPLE USAGE IN COMPONENT:
     this.apiService.get<Treatment[]>('/treatments').subscribe(
       (response) => {
         console.log(response.data);  // Handle response
       },
       (error) => {
         console.error(error);        // Handle error
       }
     );

📄 frontend/insurance-frontend/src/app/services/auth.service.ts
   LOCATION: frontend/insurance-frontend/src/app/services/auth.service.ts
   PURPOSE: Authentication state management
   KEY CODE:
   
     @Injectable({ providedIn: 'root' })
     export class AuthService {
       private tokenKey = 'auth_token';
       private roleKey = 'auth_role';
       private usernameKey = 'auth_username';
       // ... more keys ...
     
       get credentials(): string | null {
         return sessionStorage.getItem('credentials') ?? 
                localStorage.getItem('auth_credentials');
       }
     
       setCredentials(username: string, password: string): void {
         const encoded = btoa(`${username}:${password}`);
         sessionStorage.setItem('credentials', encoded);
         localStorage.setItem('auth_credentials', encoded);
       }
     
       setToken(token: string): void {
         localStorage.setItem(this.tokenKey, token);
       }
     
       setRole(role: string): void {
         localStorage.setItem(this.roleKey, role);
       }
     
       clearToken(): void {
         localStorage.removeItem(this.tokenKey);
         localStorage.removeItem(this.roleKey);
         localStorage.removeItem(this.usernameKey);
         // ... clears all auth data ...
         sessionStorage.removeItem('credentials');
       }
     
       isAuthenticated(): boolean {
         return !!this.token;
       }
     }
   
   USE:
   - Stores user authentication data (credentials, token, role, etc.) in localStorage
   - ApiService reads `credentials` property to add Authorization header
   - clearToken(): Called during logout to remove all auth data
   - SessionStorage is checked first (temporary), then localStorage (persistent)
   
   WHY TWO STORAGE LOCATIONS?
     - sessionStorage: Cleared when browser tab closes (more secure, per-session)
     - localStorage: Persists across browser sessions (remember me functionality)
     - Check sessionStorage first; fall back to localStorage for "remember me"

📄 frontend/insurance-frontend/src/app/components/login/login.component.ts

     onLogin(): void {
       const { username, password } = this.form.value;
       this.apiService.post('/auth/login', { username, password }).subscribe(
         (response: any) => {
           if (response.status) {
             this.authService.setCredentials(username, password);
             this.authService.setToken(response.token);
             this.authService.setRole(response.role);
             this.authService.setUserId(response.userId);
             this.authService.setFullName(response.fullName);
             this.authService.setEmail(response.email);
             this.router.navigate(['/dashboard']);
           } else {
             this.errorMessage = response.message;
           }
         },
         (error) => {
           this.errorMessage = 'Login failed';
         }
       );
     }
   
   USE:
   - Calls backend /api/auth/login endpoint with username/password
   - If successful, stores credentials/token/role in AuthService
   - Redirects to dashboard
   - If failed, displays error message

5. HOW TO LINK FRONTEND & BACKEND
================================================================================

METHOD 1: DURING DEVELOPMENT (Using Angular Dev Proxy)
═════════════════════════════════════════════════════════════════════════════

Step 1: Start the BACKEND
   - Navigate to project root:
     cd "Health-Insurance--Claim-Management-System-main"
   
   - Build (if needed):
     mvn clean package -DskipTests
     or
     .\mvnw.cmd clean package -DskipTests
   
   - Run:
     java -jar target\health-insurance-claim-management-0.0.1-SNAPSHOT.jar
     or right-click on: src/main/java/.../HealthInsuranceClaimManagementApplication.java → Run
   
   - Verify: Backend should start on http://localhost:8989
     Look for: "Started HealthInsuranceClaimManagementApplication in X seconds"

Step 2: Start the FRONTEND (in a NEW terminal)
   - Navigate to frontend directory:
     cd frontend\insurance-frontend
   
   - Install dependencies (if not already done):
     npm install
   
   - Start dev server:
     ng serve --open
     or
     npm start
   
   - Verify: Frontend runs on http://localhost:4200
     AutoOpen should open browser to http://localhost:4200

Step 3: How the PROXY links them:
   - You type URL or click in app: http://localhost:4200/api/treatments
   - Browser makes request to localhost:4200 (same origin)
   - Angular dev server intercepts /api/* paths
   - Proxy.conf.json redirects to: http://localhost:8989/api/treatments
   - Backend responds with JSON
   - Proxy returns response to frontend
   - Frontend JavaScript receives JSON in ApiService Observable

Step 4: Network Activity
   - Open Chrome DevTools → Network tab
   - Make any API call (login, fetch treatments, etc.)
   - You should see requests to /api/... 
   - Request payload: What you sent to backend
   - Response: JSON from backend { message, status, data }


METHOD 2: PRODUCTION BUILD (No Proxy Needed)
═════════════════════════════════════════════════════════════════════════════

Step 1: Build the FRONTEND as a static site:
   cd frontend\insurance-frontend
   ng build --configuration production

   Output: frontend/insurance-frontend/dist/insurance-frontend/ (static HTML/JS/CSS)

Step 2: Copy frontend build to backend resources:
   Copy dist/insurance-frontend/* → src/main/resources/static/

Step 3: Update backend to serve frontend:
   No code changes needed! Spring Boot serves /index.html as default page.
   All non-/api requests fall back to /index.html (for Angular routing).

Step 4: Build and run backend:
   mvn clean package -DskipTests
   java -jar target\health-insurance-claim-management-0.0.1-SNAPSHOT.jar

Step 5: Access application:
   http://localhost:8989  (server now hosts both frontend + backend)
   No proxy, no CORS issues, no separate dev server needed

Step 6: Why it works:
   - Frontend and backend are both served from same host:port (localhost:8989)
   - Same-origin policy means no CORS needed
   - All files are on same server
   - /api/* → backend API
   - /* → Angular app (static files + SPA routing)


METHOD 3: CONFIGURE FOR DIFFERENT BACKEND URL
═════════════════════════════════════════════════════════════════════════════

If you want Angular to point to a different backend URL:

Step 1: Modify src/app/services/api.service.ts:
   Change:  private baseUrl = '/api';
   To:      private baseUrl = 'http://your-backend-url.com/api';

Step 2: Remove proxy.conf.json from angular.json if not needed

Step 3: Ensure backend has CORS enabled in WebConfig.java:
   registry.addMapping("/api/**")
           .allowedOrigins("http://your-frontend-url.com")
           .allowCredentials(true)
           .allowedMethods("GET", "POST", "PUT", "DELETE", "PATCH");

Step 4: Start frontend and backend on separate hosts/ports
   Frontend: http://localhost:4200
   Backend: http://your-backend-url.com

TROUBLESHOOTING:
═════════════════════════════════════════════════════════════════════════════

Issue: "Port 8989 already in use"
   Solution: Kill process on that port or change server.port in application.properties

Issue: "Connection refused: connect to localhost:8989"
   Solution: Make sure backend is running (check Terminal 1 output)

Issue: "No credentials found for API request" (console warning)
   Solution: User not logged in yet. Call login endpoint first.

Issue: "CORS error in browser console"
   Solution: 
   - Check that proxy.conf.json is active (verify in angular.json)
   - Or add @CrossOrigin annotation to controller
   - Or enable WebConfig CORS configuration

Issue: "Proxy didn't intercept /api request"
   Solution: 
   - Make sure ng serve is running (not npm start without proxy)
   - Restart dev server after proxy changes
   - Check terminal output for proxy debug logs

1. USER INITIATES REQUEST

   this.apiService.get('/treatments').subscribe(...)

2. API SERVICE PREPARES REQUEST
   baseUrl = '/api'
   full URL = '/api/treatments'
   Headers = { Authorization: Basic <credentials>, Content-Type: application/json }

3. HTTP CLIENT SENDS REQUEST
   Browser: GET http://localhost:4200/api/treatments

4. PROXY INTERCEPTS
   Angular Dev Server proxy detects /api/* pattern
   Rewrites request to: GET http://localhost:8989/api/treatments
   Adds/modifies headers as configured (changeOrigin: true)

5. BACKEND RECEIVES REQUEST
   Spring DispatcherServlet routes to TreatmentController.getAllTreatments()
   Spring Security validates Authorization header
   If auth fails: return 401 Unauthorized
   If auth passes: proceed to handler method

6. BUSINESS LOGIC EXECUTES
   TreatmentController calls treatmentService.getAllTreatments()
   Service calls treatmentRepository.findAll()
   Repository executes: SELECT * FROM treatments
   Database returns rows

7. RESPONSE IS BUILT
   Service collects rows into List<Treatment>
   Controller wraps in response object:
     {
       "message": "Treatments fetched successfully",
       "status": true,
       "data": [
         { treatmentId: 1, name: "Dialysis", cost: 5000 },
         { treatmentId: 2, name: "Surgery", cost: 15000 },

       ]
     }

8. RESPONSE SENT BACK
   HTTP 200 OK with JSON body
   Proxy receives and forwards to browser

9. BROWSER PROCESSES RESPONSE
   ApiService Observable emits data
   Component subscription receives response
   Component.treatments = response.data
   Angular bug detection triggers
   Template re-renders with new data
   User sees treatment list

10. USER SEES DATA
    HTML divs/tables updated with treatment information
    Cycle complete!
